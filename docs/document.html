PetStore API Test Otomasyon Projesi

👤 Hazırlayan: Abdullah Aydoğan 💼 Görev: Yazılım Test ve Kalite Uzmanı 📅 Tarih: 18.05.2026

Bu doküman, Swagger PetStore API'si için geliştirilmiş olan test otomasyon projesinin teknik detaylarını, mimarisini ve kullanımını açıklamaktadır.

1. Projeye Genel Bakış

Bu proje, PetStore API'sinin temel Pet kaynağı üzerindeki CRUD (Create, Read, Update, Delete) operasyonlarının tamamını uçtan uca test eden bir otomasyon paketidir. Testler, bir evcil hayvanın yaşam döngüsünü simüle edecek şekilde sıralı bir mantıkla çalışır.

1.1. Test Akış Mimarisi

Proje, "zincirleme test" (chained tests) mantığına dayanır. Her test, bir sonrakinin çalışması için gerekli olan veriyi üretir ve bu veri, Postman ortam değişkenleri (environment variables) aracılığıyla adımlar arasında taşınır.

  1. Create Pet isteği gönderilir ve API'den dönen yanıtın içindeki id değeri, petId adıyla bir ortam değişkenine kaydedilir.
  2. Get Pet, Update Pet ve Delete Pet istekleri, bu petId değişkenini kullanarak doğru evcil hayvan kaynağı üzerinde işlem yapar.
  3. Son olarak, Verify Deletion adımı ile kaynağın kalıcı olarak silindiği teyit edilir ve test döngüsü tamamlandığında ortam değişkeni temizlenir (unset).

Bu mimari, testlerin hem bağımsız çalışabilmesini hem de birlikte anlamlı bir iş akışını test etmesini sağlar.

2. Kullanılan Teknolojiler ve Araçlar

Proje, modern API test otomasyonu pratiklerine uygun olarak aşağıdaki araç setini kullanır:

  • Postman: API isteklerini tasarlamak, test betiklerini (JavaScript) yazmak ve bunları bir koleksiyon altında gruplamak için kullanılan ana araçtır.
  • Newman: Postman koleksiyonlarını CI/CD (Sürekli Entegrasyon/Sürekli Dağıtım) süreçlerine entegre etmek veya komut satırından otomatize bir şekilde çalıştırmak için kullanılan bir CLI (Command-Line Interface) aracıdır.
  • Allure Framework: Newman tarafından üretilen test sonuçlarını, yönetici ve geliştiricilerin kolayca anlayabileceği, interaktif ve detaylı HTML raporlarına dönüştüren bir raporlama aracıdır.
  • Node.js / npm: Projenin bağımlılıklarını yönetmek ve test/raporlama betiklerini çalıştırmak için kullanılan platform ve paket yöneticisidir.

3. Proje Yapısı ve Dosyalar

Proje kök dizininde aşağıdaki önemli dosyalar ve klasörler bulunmaktadır:

  • PetStore_Collection.json: Postman'de oluşturulan ve tüm API isteklerini, test betiklerini ve dokümantasyonunu içeren ana koleksiyon dosyasıdır.
  • PetStore_Environment.json: Testlerin çalışması için gerekli olan ortam değişkenlerini (örneğin, baseUrl) barındıran dosyadır. Bu dosya, testlerin farklı ortamlarda (geliştirme, test, üretim) kolayca çalıştırılabilmesini sağlar.
  • package.json: Projenin meta verilerini (adı, versiyonu vb.) ve bağımlılıklarını (newman, allure-commandline) tanımlar. Ayrıca npm test ve npm run report gibi komutları içeren scripts bölümünü barındırır.
  • allure-results/: Newman testleri çalıştırıldığında Allure raporu için gerekli olan ham XML sonuç dosyalarının üretildiği klasördür.
  • node_modules/: npm install komutu ile yüklenen proje bağımlılıklarının bulunduğu klasördür.

4. Kurulum ve Ön Gereksinimler

Projeyi çalıştırmadan önce sisteminizde aşağıdaki araçların kurulu olduğundan emin olun:

  • Node.js (LTS versiyonu tavsiye edilir)
  • npm (Node.js ile birlikte otomatik olarak kurulur)

Kurulum adımları:

  1. Projeyi Klonlayın:
    git clone <proje-linki>
    cd PetStoreAPI
  2. Proje Bağımlılıklarını Yükleyin:
    package.json dosyasında tanımlı olan newman ve allure-commandline gibi araçları yüklemek için aşağıdaki komutu çalıştırın:
    npm install

5. Testlerin Çalıştırılması ve Raporlama

5.1. Testleri Çalıştırma

Aşağıdaki komut, package.json dosyasındaki test betiğini tetikler. Bu betik, Newman'ı kullanarak PetStore_Collection.json koleksiyonunu çalıştırır, -e parametresi ile PetStore_Environment.json dosyasını ortama dahil eder ve -r parametresi ile sonuçların hem komut satırında (cli) hem de Allure formatında (allure) üretilmesini sağlar.

npm test

5.2. Test Raporunu Görüntüleme

Testler tamamlandıktan sonra, allure-results klasöründe üretilen sonuçları kullanarak interaktif HTML raporunu oluşturmak ve görüntülemek için aşağıdaki komutu çalıştırın:

npm run report

Bu komut, allure serve komutunu çalıştırarak geçici bir web sunucusu başlatır ve oluşturulan raporu varsayılan tarayıcınızda açar.

6. Detaylı Test Senaryoları ve Doğrulamalar

Aşağıdaki tabloda, koleksiyondaki her bir isteğin amacı ve gerçekleştirdiği test doğrulamaları (assertion'lar) detaylandırılmıştır.

# İstek Endpoint / Metod Test Doğrulamaları (Assertions)
1 Create Pet POST
/pet
  • Status code is 200 OK
  • Response body'deki status alanının "available" olduğunu doğrular.
  • Dönen id'yi petId olarak ortam değişkenine kaydeder.
2 Get Pet GET
/pet/{{petId}}
  • Status code is 200 OK.
  • Dönen id'nin, kaydedilen petId ile eşleştiğini doğrular.
  • Dönen name alanının beklenen formatta (TestPet_) olduğunu kontrol eder.
3 Update Pet PUT
/pet
  • Status code is 200 OK
  • Response body'deki status alanının "sold" olarak güncellendiğini doğrular.
4 Delete Pet DELETE
/pet/{{petId}}
  • Status code is 200 OK.
  • Dönen message alanının, silinen petId'yi içerdiğini doğrular.
5 Verify Deletion GET
/pet/{{petId}}
  • Status code is 404 Not Found (Negatif Test).
  • Hata mesajının "Pet not found" olduğunu doğrular.
  • petId ortam değişkenini temizler (teardown).

7. Ortam Değişkenleri

Testlerin farklı API ortamlarında (örneğin, yerel, geliştirme, üretim) sorunsuz çalışabilmesi için PetStore_Environment.json dosyası kullanılır.

baseUrl: Test edilecek API'nin temel adresini tutar. Örnek: https://petstore.swagger.io/v2
